Git工作流

发表于 2021-03-13 更新于 2021-03-15 分类于 Git

简单记录下之前学习 Git 的过程。
最最开始，使用的是 SVN 来做版本控制，因为可以理解成只有一个版本，所以操作什么的都很简单。后来接触到 Git，也看了不少教程，但是还是比较懵逼。

从会几个简单的 Git 命令，到真正的弄明白 Git 的操作流程，是看懂了这个图。这个图包括了 Git 的所有基本用法，并且明白 Git 并不是只是学几个命令，而是一个工作流。
什么是工作流？就是从你开始写代码，到最终代码上线，这一整个流程。

Git 的工作流是什么样的？

首先 Git 把放代码的地方叫仓库。仓库分为本地仓库和远程仓库。我们写完代码，想把代码提交到远程仓库上，并不是一步到位，而是必须先把代码提交到本地仓库，然后再把本地仓库的代码提交到远程仓库上。这就是“推”代码的过程。

而“拉”代码可以一步到位，使用 Git pull origin master，直接把远程仓库的分支拉到本地。
如果有多个人合作开发，那么每个人会基于主分支，新建一个 dev 分支，待写完代码后，把 dev 分支再合并到主分支，然后推到远程仓库。什么是分支？可以理解为版本，一个版本就是一个分支。

ThinkPHP 的数据库中间层架构

发表于 2021-03-13 更新于 2021-03-15 分类于 ThinkPHP

一、代码操作数据库的核心

操作数据库的过程核心有两个步骤：

  • 连接数据库
  • 操作数据库

如果只有一种数据库，比如 Mysql，那么直接用面向过程的思路，一步步编写下去即可。但是连接数据库和操作数据库，一定会有的需求：需要连接不同的数据库，并且不同的数据库的操作方法是不一样的。如果按照面向过程的思想，每一个数据库都需要单独写一套连接数据库和操作数据库的方法，会非常麻烦。

这个时候，使用面向对象的封装，就可以很好的解决这个问题。把连接数据库和操作数据库的方法进行封装，不管是连接什么数据库还是这个数据库使用什么的操作方法，都使用同一套方法，把不同数据库的差异性，封装起来。TP 的数据层和数据连接层使用的就是这个方法。

二、TP 数据层的具体实现

Db 是数据库操作的入口，TP 里面所有关于数据库的操作，都是使用 Db 这个类来实现的，这个类在 thinkphp/library/Db.php 这个文件里面。
比如 TP 里面执行某条 SQL 语句，会使用这种方式。
$result = Db::query(‘select * from banner_item where banner_id=?’,[$id]);

TP 里面数据库的实现，主要是依靠下面这些文件，都在 db 这个目录下。

它们之间的关系：如下图。

Connection：用来连接数据库和执行 SQL。Connection 的下一层是 connector，connector 具有不同数据库的连接类（如下图），connection 根据 database.php 里面的配置，调用 connector 不同的数据库连接类，就可以实现对连接不同数据库的封装。

Builder：用来将 Query 查询器的标准化 API，转换成原生 SQL。转换成 SQL 之后，再调用 Connection 来执行，也就是上面的过程。Builder 下面也有很多不同的文件，用于执行不同的 SQL。

以上，就是 TP 中，解决使用同一套方法，来实现连接和操作不同数据库的过程。
不得不说，面向对象太强了！！
这其实是一个从具体到抽象的过程，代码的抽象程度越高，适用的范围也就越广。以前会奇怪，为什么有的代码写的这么绕，为什么明明每行代码单独都看的懂，业务逻辑也明白，但是整体看就是看不懂，其实就是因为代码的抽象程度很高，需要从更高的维度去理解。学到了！！

今天遇到一个比较奇怪的问题，如下图。一直提示找不到 driver。

Google 了一下，发现解决办法就是修改 php.ini，把下面这行前面的“；”给去掉。尝试之后，发现无效。
extension=pdo_mysql

推测一：没有把 mysql 的 pdo 包给导入进来

验证：在命令行输入 php -m，可以看到 pdo_mysql 已经导入了。

推测二：php.ini 没有生效

又疯狂 Google 后，终于找到了一个解决办法。修改 php.ini 文件里面的 extension_dir 为绝对路径。

原因应该是，tp 框架顺着拓展路径去找包，但是因为路径问题，没有找到。修改一下路径就好了。

彻底解决VisualSutdio中文乱码的问题

解决办法

fromLocal8Bit 的作用是使 GBK/GB2312/GB18030 格式的字符串转换成 Unicode 格式。

新出现的问题

过了一段时间，结果按照上面的方式写，又出现中文乱码,如下图。理论上来说，同样的代码执行结果应该是完全一样的。所以可能不是代码的问题，而是别的问题。

原因

经过排查和网上搜索相关的文章，发现问题的原因果然不在代码上，而文件的保存格式有问题。通常代码里面中文的编码格式会设置成 UTF-8。但是 UTF-8 也分为两种，一种带签名，一种不带签名。带签名的会有 BOM 信息，来告诉应用程序所用的是 UTF-8 编码。

  • UTF-8（带签名）：带有 BOM 信息，可以被应用程序识别为 UTF-8 编码。
  • UTF-8（不带签名）：不带有 BOM 信息，不能被应用程序识别为 UTF-8 编码。在被其他文件 include 时，需要使用这种。

因为编码问题而出现乱码的情况，无非也就两种：一种是在应该使用 UTF-8（带签名）的文件里使用不带签名的编码。另外一种是在不应该带签名的文件里面，使用了带签名的 UTF-8 编码，使文件有两层 BOM 信息，从而导致乱码。

上面问题出现的原因，是因为这个文件没有带 BOM 信息，导致应用程序无法识别，而出现乱码。

解决办法

  1. 在 VS 里，添加高级保存选项。
    A.选择工具 → 自定义 → 打开命令 → 菜单栏选择【文件】

B.点击【添加命令】，选中【文件】，选择【高级保存选项】。

2.在菜单栏里，点击文件，打开高级保存选项。选择编码格式为【Unicode（UTF-8 带签名）】，点击确定。
3.修改完成后，中文显示正常，如下图。中文乱码问题解决。

闲谈：计算机和语言

发表于 2020-05-01 更新于 2021-03-15 分类于 随笔

在计算机中，我们经常会碰到精度丢失的问题，比如 100/3=33.33，那么会有 0.01 的精度被丢失掉了。之前以为这种是程序设计的问题，但是后来发现，这个问题的根源，不在于计算机，而在于与计算机紧密相连的数学问题。是数学规定，100/3，就是除不尽。这种问题，可以通过程序设计来规避，比如支付宝的 AA 收款，如果有 100 元，三个人均摊，那么发起 AA 收款的人，需要支付 33.34，另外两个人需要支付 33.33。

另外一种精度问题，是属于计算机的设计缺陷。比如在计算机里，0.1+0.2！=0.3，因为计算机在处理类似等式时，会把他们换成二进制数，再进行相加。这类问题的根源，在于计算机设计时，采用了二进制的计算方法。这种问题，也可以通过程序设计来进行规避。

由此想到了语言本身的缺陷。如果一个语言系统足够完美，那么它是不会出现歧义的问题。语言会有歧义，说明语言本身有缺陷。这些缺陷，对于语言来说，并不一定都是坏事，诗人会利用歧义来表达更深刻的意义，幽默的人会利用歧义来表达幽默，歧义在一定程度上，让语言更易于学习。因为一门完美的语言，必定会设计过多的规则去限制，从而导致语言的复杂性变高，更难学习。

相比于中文，我觉得英文的设计，更加简洁和有逻辑。比如时态这种东西，中文里并没有严格意义上的时态体系，中文一般是通过给动词加状语来表示时态。但是英文里面，光时态的划分就有 16 种。

更有趣的是，我们可以自己尝试去设计一门语言，用自己制定的规则，去和中文、英文做比较，一是可以体会到造物主的乐趣，毕竟不是所有人都有创造语言的经历，二是可以更好的去学习英语。假如我们学习英语语法，不再是去死记硬背英语的语法，而是去比较英语比我自己设计的语言，好在哪里？为什么要这么设计？有哪些可以借鉴的地方？通过对比，我们可以更深刻的去理解语言背后的逻辑是什么。

语言本身也存在缺陷，完美的语言系统，是不会出现歧义的问题。相比于中文，英文的设计更加简洁和优雅。中文会显得繁琐，并缺少逻辑。比如表示时态，英文光时态就有十六种时态，而中文是通过给动词加状语，来表示时间。比如，中文说他去过学校，英文按照时态，可以分成： 4.我好像分不清锦和绵。

浏览器缓存

发表于 2019-09-21 更新于 2021-03-15 分类于 前端

什么是浏览器缓存？

为了节约网络的资源，浏览器在用户的磁盘上对最近请求过的文档进行存储。当再次访问时，就可以直接从本地磁盘中获取，这样就可以加速页面的阅览，并且缓解一部分服务器的压力。
浏览器缓存的缓存机制是通过 Http 协议来实现的。

浏览器缓存存在哪？

浏览器缓存的位置可以分为四种：
1.Service Worker
2.Memory Cache
3.Disk Cache
4.Push Cache
