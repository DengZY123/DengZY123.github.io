Laravel学习笔记（一）路由

发表于 2021-04-10 更新于 2021-07-14 分类于 Laravel

一、安装 Laravel

1.配置环境

可以直接下载 PHPstudy，进行 Nginx 和 Mysql 的配置。

2.新建 Laravel 项目

建议安装 composer，可以直接使用 composer 新建 Laravel 项目。安装过程可以自行百度，这里不再赘述。
输入命令：
    composer create-project laravel/laravel new_project --prefer-dist "5.*"

composer create-project 后面一共有四个参数，分别为：

  • laravel/laravel：找到 Laravel 目录下的 Laravel 包
  • new_project：新建项目的名称
  • –prefer-dist：优先使用本地的包
  • “5.*“ ：指定下载版本

至此，Laravel 项目已经建好。

二、Laravel 的路由

1.基础的路由

Laravel 最简单的路由有两种方式，一种是通过 URL+闭包的形式，如下：
Route::get('/user', function () { return 'Hello World'; }); 
另外一种是通过 URL+controller 的方式，如下：
Route::get('/user', 'UserController@index'); 
两种方式都可以通过：http://your-app.dev/user，来访问

2.增强版路由

2.1 加参数

路由参数的形式有两种，一种是必传参数，一种是可传参数
必传参数：
在 URL 中写入{id}即可，{id}可以作为闭包函数的参数来使用。如果想要传多个参数，可以多加几个{var}。
Route::get('user/{id}', function ($id) { return 'User '.$id; });  
可传参数：
可选参数是在必传参数的后面加一个？来实现。如果要在闭包函数里面使用，需要给参数设定一个默认值。
Route::get('user/{name?}', function ($name = 'John') { return $name;});

2.2 对参数的约束

Laravel 对参数的约束主要依靠正则表达式来完成，只有当 name 为字母时，才会匹配到这个路由。
Route::get('user/{name}', function ($name) { })->where('name', '[A-Za-z]+');

2.3 路由组

路由组可以让某些路由，拥有相同的属性，比如中间件、命名空间、路由前缀等。

购物车+结算业务逻辑梳理

发表于 2021-03-15 更新于 2021-07-14 分类于 随笔

电商里面都会有类似加入购物车，然后结算的业务。一般是会属于系统里面非常复杂的模块，所以想在这里，从业务逻辑的角度梳理一下。
这条业务完整的逻辑是：选择商品 → 加入购物车 → 订单结算 → 结算成功。
如果走一切正常的逻辑，还是挺简单的，无非是在数据库里面加一条订单结算的记录。但是复杂的地方在于：商品和结算。

商品

电商平台中的商品，会有许多属性，包括：商品名、价格、单位、库存、产地等等。对商品的操作也有很多，包括：修改商品名、修改价格、变更库存、上/下架。商品的属性和操作都会影响到订单结算，所以在订单结算前，需要对待结算单里面的商品，进行多重判断，导致整个业务逻辑非常复杂。

结算

小程序进行微信支付的逻辑如下图。可以简单的理解为：客户端生成支付订单 → 服务器调用微信支付系统 → 微信支付系统生成预付单给服务器 → 客户端获取预付单的信息 → 客户完成支付 → 服务器异步返回支付结果。

其中，比较容易出问题的点在于：客户支付可能成功，也可能失败（比如客户主动取消了支付）；微信支付系统是异步返回结果给服务器，所以不是调用支付完就可以立马得到支付结果。

ThinkPHP 的数据库中间层架构

发表于 2021-03-13 更新于 2021-07-14 分类于 ThinkPHP

一、代码操作数据库的核心

操作数据库的过程核心有两个步骤：

  • 连接数据库
  • 操作数据库

如果只有一种数据库，比如 Mysql，那么直接用面向过程的思路，一步步编写下去即可。但是连接数据库和操作数据库，一定会有的需求：需要连接不同的数据库，并且不同的数据库的操作方法是不一样的。如果按照面向过程的思想，每一个数据库都需要单独写一套连接数据库和操作数据库的方法，会非常麻烦。

这个时候，使用面向对象的封装，就可以很好的解决这个问题。把连接数据库和操作数据库的方法进行封装，不管是连接什么数据库还是这个数据库使用什么的操作方法，都使用同一套方法，把不同数据库的差异性，封装起来。TP 的数据层和数据连接层使用的就是这个方法。
