闲谈：计算机和语言

发表于 2021-05-01 更新于 2021-03-13 分类于 随笔

在计算机中，我们经常会碰到精度丢失的问题，比如 100/3=33.33，那么会有 0.01 的精度被丢失掉了。之前以为这种是程序设计的问题，但是后来发现，这个问题的根源，不在于计算机，而在于与计算机紧密相连的数学问题。是数学规定，100/3，就是除不尽。这种问题，可以通过程序设计来规避，比如支付宝的 AA 收款，如果有 100 元，三个人均摊，那么发起 AA 收款的人，需要支付 33.34，另外两个人需要支付 33.33。

另外一种精度问题，是属于计算机的设计缺陷。比如在计算机里，0.1+0.2！=0.3，因为计算机在处理类似等式时，会把他们换成二进制数，再进行相加。这类问题的根源，在于计算机设计时，采用了二进制的计算方法。这种问题，也可以通过程序设计来进行规避。

由此想到了语言本身的缺陷。如果一个语言系统足够完美，那么它是不会出现歧义的问题。语言会有歧义，说明语言本身有缺陷。这些缺陷，对于语言来说，并不一定都是坏事，诗人会利用歧义来表达更深刻的意义，幽默的人会利用歧义来表达幽默，歧义在一定程度上，让语言更易于学习。因为一门完美的语言，必定会设计过多的规则去限制，从而导致语言的复杂性变高，更难学习。

相比于中文，我觉得英文的设计，更加简洁和有逻辑。比如时态这种东西，中文里并没有严格意义上的时态体系，中文一般是通过给动词加状语来表示时态。但是英文里面，光时态的划分就有 16 种。

更有趣的是，我们可以自己尝试去设计一门语言，用自己制定的规则，去和中文、英文做比较，一是可以体会到造物主的乐趣，毕竟不是所有人都有创造语言的经历，二是可以更好的去学习英语。假如我们学习英语语法，不再是去死记硬背英语的语法，而是去比较英语比我自己设计的语言，好在哪里？为什么要这么设计？有哪些可以借鉴的地方？通过对比，我们可以更深刻的去理解语言背后的逻辑是什么。

语言本身也存在缺陷，完美的语言系统，是不会出现歧义的问题。相比于中文，英文的设计更加简洁和优雅。中文会显得繁琐，并缺少逻辑。比如表示时态，英文光时态就有十六种时态，而中文是通过给动词加状语，来表示时间。比如，中文说他去过学校，英文按照时态，可以分成： 4.我好像分不清锦和绵。

ThinkPHP 的数据库中间层架构

发表于 2021-03-13 分类于 ThinkPHP

一、代码操作数据库的核心

操作数据库的过程核心有两个步骤：

  • 连接数据库
  • 操作数据库

如果只有一种数据库，比如 Mysql，那么直接用面向过程的思路，一步步编写下去即可。但是连接数据库和操作数据库，一定会有的需求：需要连接不同的数据库，并且不同的数据库的操作方法是不一样的。如果按照面向过程的思想，每一个数据库都需要单独写一套连接数据库和操作数据库的方法，会非常麻烦。

这个时候，使用面向对象的封装，就可以很好的解决这个问题。把连接数据库和操作数据库的方法进行封装，不管是连接什么数据库还是这个数据库使用什么的操作方法，都使用同一套方法，把不同数据库的差异性，封装起来。TP 的数据层和数据连接层使用的就是这个方法。

二、TP 数据层的具体实现

Db 是数据库操作的入口，TP 里面所有关于数据库的操作，都是使用 Db 这个类来实现的，这个类在 thinkphp/library/Db.php 这个文件里面。
比如 TP 里面执行某条 SQL 语句，会使用这种方式。
$result = Db::query(‘select * from banner_item where banner_id=?’,[$id]);

浏览器缓存

发表于 2019-09-21 更新于 2021-03-13 分类于 前端

什么是浏览器缓存？

为了节约网络的资源，浏览器在用户的磁盘上对最近请求过的文档进行存储。当再次访问时，就可以直接从本地磁盘中获取，这样就可以加速页面的阅览，并且缓解一部分服务器的压力。
浏览器缓存的缓存机制是通过 Http 协议来实现的。

浏览器缓存存在哪？

浏览器缓存的位置可以分为四种：
1.Service Worker
2.Memory Cache
3.Disk Cache
4.Push Cache

Service Worker

自由控制缓存哪些内容，如何缓存，何时读取。
出于安全考虑，只能在 Https 或者本地 localhost 环境下使用
我们可以通过谷歌开发者工具中的 Application 查看当前的缓存资源。

Mermory Cache

内存中的缓存，其特点是容量小，读取高效，持续性短
在内存使用频率低，缓存小尺寸资源时，会优先使用 Mermory Cache

Disk Cache

容量大，读取缓慢，持续性长。
在内存使用频率高，缓存大尺寸资源时，会优先使用 Disk Cache

Push Cache

只在会话（session）中存在
会话结束就释放，存在时间很短暂。

浏览器缓存有哪几种？

目前浏览器的缓存有两种，一种是强制缓存，一种是协商缓存。

强制缓存

强制缓存：不会向服务器发送请求，而是直接从缓存中读取资源。在 chrome 浏览器中，可以看到的强制缓存，如图。状态码为 200，size 显示的 memory cache 或 disk cache。

强制缓存的主要情况有三种，如下：

不存在缓存的结果和缓存标识，强制缓存失效，直接从服务器请求。

存在缓存标识和缓存结果，但是缓存结果已失效。这时，强制缓存失效，使用协商缓存。

存在缓存结果和缓存标识，且缓存标识未失效，则直接从浏览器读取缓存结果。

那么，浏览器是如何知道哪些缓存标识失效的呢？
这里主要用到的是 HTTP 请求的 header 中的两个字段，Expires和Cache-control。
Expires：缓存过期时间，用来指定资源到期的时间。再次发送请求时，如果客户端的时间小于 Expires，则重新发送请求。
Cache-control：HTTP/1.1 中，最重要的规则，主要用来控制网页缓存。
